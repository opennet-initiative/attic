Das olsr-Programm (der olsr-Daemon) ist für die Berechnung und Verwaltung der Routen, also der Wege im Funknetz zuständig. An dieser Stelle können einige ausgewählte Erweiterungen (plugins) aktiviert bzw. deaktiviert werden. Eine weitergehende Konfiguration ist hier nicht möglich.
Da die meisten Optionen dieses Programs nur netzweit sinnvoll konfigurierbar sind, wird empfohlen, die automatische Synchronisation der Einstellungen zu aktivieren. Dabei wird stündlich geprüft, ob eine aktualisierte Konfigurationsdatei auf dem Gateway vorliegt und diese wird dann auf den AccesPoint kopiert. Konfigurationsoptionen in dieser Datei haben Priorität gegenüber der auf dem Accesspoint vorhandenen olsrd.conf_initial.
Wurde die Konfiguration der plugins oder die Konfiguration auf dem Gateway geändert, wird das olsr-Programm neu gestartet. Dies geschieht nicht unmittelbar, sondern etwa mit (maximal) einer Minute Verzögerung.
